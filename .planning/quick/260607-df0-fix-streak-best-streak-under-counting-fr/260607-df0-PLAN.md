<objective>
Fix streak and best-streak under-counting caused by an unsorted `dlng_history` array.

`recordGame` prepends solves in play order (deduped by date but not sorted). `computeStats`
walks the array assuming it is sorted newest-first, so a single out-of-order entry creates a
false early gap that splits one real consecutive run into a too-small current streak and a
wrong best streak. Real evidence: a 25-day unbroken run displayed as streak 5 / best 16.

Purpose: stats numbers must reflect the player's actual streaks. This heals already-broken
stored histories on read (no data migration) and keeps future writes clean.

Output: corrected `computeStats` (sort a copy before walking), corrected `recordGame`
(sort on write), and regression tests covering the real-world jumbled case.

Scope guard: numbers only. Do NOT change displayed metrics, labels, layout, or worker/API.
ES2022 target. Frontend-only.
</objective>
