## Celebration Animation Contract

### Timing breakdown — current vs target

| Phase | Current | Target | Note |
|-------|---------|--------|------|
| Lead-in (header → centre) | 350ms | 200ms | ease-in |
| Fly animation (`octo-fly`) | 5000ms | 2000ms | cubic-bezier(0.4, 0, 0.6, 1) |
| Return (centre → header slot) | 650ms | 400ms | ease-in-out |
| **Total** | **6000ms** | **2600ms** | Under 3s target |

**`LEAD_IN_MS` constant:** Change from 350 to 200.
**`octo-fly` duration:** Change from `5s` to `2s` in `style.css` `.celebrating` rule.
**Return transition:** Change from `0.6s` to `0.4s` in `octo.ts` `celebrateOcto()`.
**`setTimeout` that triggers return:** Change from `5100 + 350` (5450ms) to `2000 + 200` (2200ms).
**`TOTAL_MS` in `bubbles.ts`:** Change from 6500 to 3200 (slightly longer than animation so bubbles can finish rising).
**Bubble individual durations (`randomDuration`):** Change from `4100–5000ms` to `2000–2600ms`.

### Skip interaction

- Register a single `{ once: true }` click/touchstart listener on `document.body` at the start of `celebrateOcto()`.
- On skip: cancel in-flight timeouts and transitions, snap octo back to header slot (no return animation — instant position restore), then call the completion callback immediately.
- Clean up the body listener if the animation completes naturally (remove before calling the callback).
- Use a cleanup function pattern: `const cleanup = () => { document.body.removeEventListener(...) }`. Call cleanup in both the natural-end path and the skip path.

### Completion callback

`celebrateOcto()` accepts an `onComplete: () => void` callback. The caller (`app.ts` correct-answer handler) passes `() => showScreen('completion')`. Both the natural end path and the skip path call this callback.

### Reduced motion

Check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` in the correct-answer handler before calling `celebrateOcto()`.

- If reduced motion is active: skip `celebrateOcto()` and `launchBubbles()` entirely. Call `showScreen('completion')` directly.
- `launchBubbles()` already guards itself — no change needed there.
- Cross-fade transitions (opacity) are kept even under reduced motion (they are not vestibular motion).

Source: D-01 through D-04, D-13, D-14 from `05-CONTEXT.md`; `src/octo.ts` celebrateOcto() lines 243–319; `src/bubbles.ts` TOTAL_MS.

## Stats Grid Contract

Four boxes in a 2×2 grid. Each box: `bg-surface border border-border rounded-md p-4 text-center`.

| Box | Number source | Label |
|-----|--------------|-------|
| Top-left | `history.length` | Played |
| Top-right | `(total tries / count).toFixed(1)` | Avg tries |
| Bottom-left | Computed consecutive streak ending today | Streak |
| Bottom-right | Max consecutive streak across all history | Best streak |

**Streak computation rule:** Walk `loadHistory()` entries (sorted newest-first by date). Count consecutive calendar days from today backwards. A gap of more than 1 day resets the current streak count. Max streak is the longest run found in any contiguous block.

No "win %" stat — always 100% in Clumeral. Source: D-05, D-06, D-07 from `05-CONTEXT.md`.

## Countdown Timer Contract

Format: `"Next puzzle in Xh Ym"` — hours and minutes only. No seconds ticking.

- Compute time until next midnight (local timezone) at render time.
- Display is static — computed once when completion screen appears. No live update tick.
- If less than 1 hour remaining: `"Next puzzle in Ym"` (omit hours).
- If puzzle is a random puzzle (not daily): hide the countdown entirely.

Example values: `"Next puzzle in 4h 23m"`, `"Next puzzle in 51m"`.

Source: D-10 from `05-CONTEXT.md`; ENH-01 pulled into v1 scope.
