## Color

All colours reference the 6 semantic Tailwind tokens defined in src/tailwind.css @theme.

| Role | Token | Light value | Dark value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | bg-bg | #FAFAFA | #121213 | Page background behind backdrop |
| Secondary (30%) | bg-surface | #FFFFFF | #363634 | Modal card background; textarea background; pill (unselected) background |
| Accent (10%) | text-accent / bg-accent | #0A850A | #1EAD52 | Selected pill fill; textarea focus ring; counter warn colour |
| Text | text-text | #262624 | #FAF8F4 | Heading, pill labels, textarea input, close button |
| Muted | text-muted | rgba(38,38,36,0.7) | rgba(246,240,232,0.6) | Subtitle, metadata text, counter default, pill placeholder |
| Border | border-border | rgba(38,38,36,0.12) | rgba(246,240,232,0.1) | Pill borders (unselected), textarea border |

Additional values needed for this phase (not in the 6 tokens — use raw values):
- Backdrop: rgba(0,0,0,0.55) — semi-transparent dark scrim behind modal card
- Modal card shadow: light: `0 8px 40px rgba(26,24,48,0.18)` / dark: `0 8px 40px rgba(0,0,0,0.5)` — use Tailwind arbitrary shadow or CSS variable
- Toast bg: bg-text (inverted — text colour as background, bg colour as text). This creates automatic light/dark inversion.
- Send button bg: accent (#0A850A fixed — not theme-aware, always dark green for WCAG contrast against white text)
- Send button shadow: `3px 3px 0 rgba(38,38,36,0.25)` (offset press effect)
- Metadata block bg: accent at ~10% opacity (`rgba(10,133,10,0.08)` light / `rgba(30,173,82,0.1)` dark) — same as legacy `--md-lit-bg`

Accent is reserved for:
- Selected category pill fill + border
- Textarea border when focused
- Character counter text when ≥ 400 characters
- Send button background

Destructive: none in this phase. No destructive actions.

## Component Inventory

### Modal dialog (`<dialog data-fb-modal>`)
- Full-viewport `<dialog>` element, border none, background transparent, padding 16px
- Opens with `showModal()`, closes with `close()` (existing modals.ts pattern — do not change)
- Backdrop: rgba(0,0,0,0.55) via `::backdrop` pseudo-element
- Open/close animation: opacity 0→1 over 200ms + modal card translateY(12px)→translateY(0) over 200ms (fade + slight scale from D-03; translate replaces scale to avoid layout reflow)
- `aria-labelledby="cw-fb-modal-title"` (must match heading id)

### Modal card (`[data-fb-modal-box]`)
- Max width: 26rem (416px) centred
- Max height: 90vh with overflow-y auto
- Border radius: 6px (rounded-md)
- Padding: 28px top, 24px sides, 24px bottom
- Background: surface token
- Shadow: modal card shadow values above

### Close button (`[data-fb-modal-close]`)
- Position: absolute, 16px from top, 16px from end
- Min size: 44px × 44px touch target
- Style: no background, no border, text-text at 50% opacity
- Hover: opacity 100%
- Content: "✕" character (existing) or SVG ×-icon from sprites.svg — either is acceptable

### Category pills (`[data-fb-cats]` + `.fb-cat`)
- Container: flex row, flex-wrap, gap 8px, margin-bottom 12px
- Pill: rounded-full (border-radius 9999px)
- Unselected: border 1.5px border-border, bg-surface, text-text
- Selected (`aria-checked="true"`): bg accent (#0A850A), border accent, text white
- Padding: 6px 14px
- Font: 16px / 400
- Transition: background-color, color, border-color at 150ms
- ARIA: `role="radiogroup"` on container, `role="radio"` + `aria-checked` on each pill

### Textarea (`[data-fb-msg]`)
- Full width, min-height 96px, max-height 192px, resize vertical
- Border: 1.5px border-border
- Border on focus: 1.5px accent colour, outline none
- Background: bg-surface, text-text, placeholder text-muted
- Padding: 12px
- Border radius: 4px
- Font: DM Sans 16px / 400 / line-height 1.5
- maxlength="500" (enforced in HTML)

### Character counter (`[data-fb-counter]`)
- Inline size: full width, text-align end
- Font: Inconsolata 16px / 400
- Hidden when < 400 chars (Tailwind `hidden` class or `sr-only` pattern)
- Shows "NNN/500" when ≥ 400 chars
- Colour: text-muted by default, text-accent when ≥ 400 (warn state)
- Margin-top: 4px

### Metadata block (`[data-fb-meta]`)
- Label above block: "This info will be sent with your feedback:" — text-muted 16px
- Block: Inconsolata 16px / line-height 1.6 / text-muted
- Background: accent at ~10% opacity (see colour section)
- Padding: 8px 12px
- Border radius: 4px
- Margin-top on label: 12px; margin-bottom on label: 6px

### Send button (`[data-fb-send]`)
- Full width
- Height: 48px
- Background: #0A850A (fixed green — not theme-aware)
- Text: white, 16px, weight 600, letter-spacing 0.04em
- Border: none
- Border radius: 3px
- Shadow: 3px 3px 0 rgba(38,38,36,0.25)
- Active: translateY(2px) translateX(2px), shadow none (press effect)
- Disabled: opacity 50%, cursor not-allowed
- Margin-top: 12px

### Toast (`[data-toast]` container + dynamically created elements)
- Container: fixed, bottom 24px, horizontally centred, z-index 300
- Toast element: `div` created by `showToast()` in modals.ts — class replaced from `toast__msg` to Tailwind utilities
- Padding: 12px 20px
- Border radius: 6px
- Background: bg-text token (inverted — automatic light/dark)
- Text: bg-bg token (inverted)
- Font: DM Sans 16px / 500
- Shadow: 0 4px 16px rgba(0,0,0,0.2)
- Appear animation: opacity 0→1 + translateY(8px)→0 over 250ms
- Auto-dismiss: 3000ms default (existing `showToast` duration param — do not change)
- Exit: opacity 1→0 + translateY back, then element removed via transitionend

Source: src/style.css toast block; CONTEXT.md D-04, D-05.

## Interaction Contract

### Open modal
- Trigger: `[data-fb-btn]` (game menu) or `[data-fb-header-btn]` (completion screen — Phase 5)
- Effect: `showModal()` + addClass "open" on next animation frame
- Backdrop and card fade in simultaneously over 200ms

### Close modal
- Triggers: close button click, Escape key (native `<dialog>` behaviour)
- Effect: removeClass "open", `close()` called on transitionend
- Do not close on backdrop click (no backdrop click handler — matches existing behaviour)

### Category pill selection
- Click: set aria-checked true on clicked pill, false on all others; update `selectedCat`; apply selected styles
- Keyboard: Space/Enter on focused pill selects it (button element handles this natively)

### Character counter
- On textarea input: if length ≥ 400, show counter with "NNN/500" and apply warn colour; if < 400, hide counter
- No user action required — purely reactive to input event

### Submit
- Click send button: disable button, set `sending = true`, POST to Google Apps Script endpoint
- Success: close modal, `showToast("Thanks for your feedback!")`, reset form
- Retry on failure: up to 2 retries with 1s delay (existing retry logic in modals.ts — do not change)
- Final failure: `showToast("Something went wrong. Please try again.")`, re-enable button

### Form reset
- On modal close (after transitionend): reset textarea, reset pill to "general" selected, hide counter, re-enable send button

## Accessibility Contract

- `<dialog>` with `showModal()` traps focus automatically (browser native)
- `aria-labelledby` on dialog references heading id `cw-fb-modal-title`
- Category pills: `role="radiogroup"` / `role="radio"` / `aria-checked`
- Textarea has `<label for="cw-fb-msg">` (sr-only) so screen readers announce it
- Close button has `aria-label="Close"`
- Focus returns to the trigger element that opened the modal when closed (modals.ts must save and restore focus — verify this is already handled)
- Colour contrast: accent (#0A850A) on white meets WCAG AA for normal text; white on #0A850A send button meets WCAG AA
- Counter warn colour: accent on surface — check meets WCAG AA (currently borderline; acceptable as it is supplementary to the visible counter text)
