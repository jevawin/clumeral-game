    Create `tests/setup.ts` with shared test utilities. It must:
    - Reset `localStorage` between tests via `beforeEach(() => localStorage.clear())` from `vitest`.
    - Stub `fetch` to a `vi.fn()` that resolves to `{ ok: true, json: async () => ({}) }` so `track()` calls in app code do not error.
    - Reset `document.title` to `'Clumeral'` between tests.
    - Reset `history.scrollRestoration` to `'auto'` between tests where the property is writable.

    Exact content for `tests/setup.ts`:

    ```ts
    import { beforeEach, vi } from 'vitest';

    beforeEach(() => {
      localStorage.clear();
      document.title = 'Clumeral';
      try {
        if ('scrollRestoration' in history) history.scrollRestoration = 'auto';
      } catch {
        /* read-only in some envs — ignore */
      }
      vi.stubGlobal('fetch', vi.fn(async () => ({ ok: true, json: async () => ({}) }) as unknown as Response));
    });
    ```

    Use kebab-case filenames (CLAUDE.md). All imports use the `.ts` extension where required by project rules — vitest resolves bare specifiers fine, so no `.ts` on `vitest` or `vitest/config`.
