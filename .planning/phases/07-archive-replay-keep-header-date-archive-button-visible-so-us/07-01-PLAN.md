<objective>
Fix issue #208: the archive date banner and Archive button must stay visible the entire time a
user is on /play for an archive puzzle — before, during, and after solve — so they always have an
exit without being forced to solve.

Two-part frontend-only fix, both in `src/app.ts`:

1. **Brand-button routing (LOCKED DECISION):** When on an `/archive/<date>` URL, the brand
   (Clumeral logo) navigates to `/archive` (the archive list) instead of `/play`. This is the
   documented archive exit per docs/URL-ARCHITECTURE.md and removes the stranding bug at its root.

2. **Archive-row visibility anchor:** Stop hiding `[data-archive-row]` based on `location.pathname`
   in the `route_change` analytics handler. Anchor archive-mode to the canonical
   `gameState.date && gameState.date !== todayKey()` signal so URL/state divergence can no longer
   hide the row mid-session, and so the row survives the solve transition.

Purpose: Restore a reliable exit affordance for archive replay; eliminate the broken
URL-as-archive-flag coupling.
Output: Updated `src/app.ts`, updated `docs/URL-ARCHITECTURE.md`.
</objective>
