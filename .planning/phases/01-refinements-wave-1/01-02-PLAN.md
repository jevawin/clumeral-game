    The new `menu-item` class is the hook for the CSS hover rule below. Existing utilities (flex, padding, etc.) stay so the layout does not change.

    **C. `src/tailwind.css` — add menu hover rule (MNU-03).**

    Inside `@layer base` (the existing layer block ending around line 286), add a new section after the `.burger-btn` rules (around line 217, before the `[data-save]` block). Insert:

    ```css
    /* ─── Burger menu items (MNU-02 / MNU-03) ─── */
    /* No hover background; text + icon both turn accent green on hover. */
    [data-menu] .menu-item {
      background-color: transparent;
      transition: color 150ms ease;
    }
    [data-menu] .menu-item:hover,
    [data-menu] .menu-item:focus-visible {
      background-color: transparent;
      color: var(--color-accent);
    }
    [data-menu] .menu-item svg {
      color: inherit;
    }
    @media (prefers-reduced-motion: reduce) {
      [data-menu] .menu-item { transition: none; }
    }
    ```

    Why `color: inherit` on the SVG: the icons use Lucide-style `<use href="/sprites.svg#icon-...">`. With `text-text` on the SVG and `currentColor` in the sprite paths (verify via `grep "currentColor" public/sprites.svg`), the icon picks up the parent `.menu-item`'s `color`. On hover, the parent goes accent → icon goes accent. If the sprite paths use a fixed fill (not currentColor), the icon will not change color — that is acceptable for v1.1; the text color change alone satisfies the "text changes to accent green" requirement and is the primary visual hover signal.

    **D. Sanity check sprite use of currentColor.**

    Run `grep -c 'currentColor' public/sprites.svg`. If returns 0, accept that icon color does not transition on hover for this plan; the text-color transition still meets MNU-03's letter ("change text color to the accent green"). Do NOT modify `public/sprites.svg` in this plan.

    - Don't touch `initMenu()` in `src/app.ts`.
    - Don't change the dropdown container's existing classes (`bg-surface border border-border rounded-lg shadow-lg` at line 163) — those define the menu panel itself, not item hover.
  </action>
