## Color

Six semantic tokens defined in `src/tailwind.css` `@theme`. Light and dark mode values are both declared. The executor must use only these tokens — no hardcoded hex values in migrated HTML or CSS, except where the token system itself defines them (keyframe colours).

| Role | Light value | Dark value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) `--color-bg` | `#FAFAFA` | `#121213` | Page background, screen backgrounds |
| Secondary (30%) `--color-surface` | `#FFFFFF` | `#363634` | Cards, modal box, keypad button surfaces |
| Accent (10%) `--color-accent` | `#0A850A` | `#1EAD52` | Listed below — accent reserved-for list |
| Text `--color-text` | `#262624` | `#FAF8F4` | All body and heading text |
| Muted `--color-muted` | `rgba(38,38,36,0.7)` | `rgba(246,240,232,0.6)` | Secondary/helper text, metadata, clue labels |
| Border `--color-border` | `rgba(38,38,36,0.12)` | `rgba(246,240,232,0.1)` | All borders, dividers, digit box outlines |

Destructive: no destructive color token exists. Phase 6 has no destructive actions.

**Accent (`--color-accent`) reserved for:**
- Submit button background and focus ring
- Correct-guess digit box background tint and border
- Feedback category pill selected state (background + border)
- Character counter `.warn` state
- Octo animation keyframe colour cycling start/end (`octo-colours`)
- Bubble fill colour in `bubbles.ts`
- Logo SVG fill on welcome screen (`var(--color-accent)`, replacing old `var(--acc)`)
- Any interactive focus ring that needs a visible accent

Do NOT apply accent to: body text, muted text, borders in default state, surface backgrounds, heading text.

Source: `src/tailwind.css` `@theme` and `@layer base` (dark overrides); CONTEXT.md D-01, D-02; REQUIREMENTS.md STY-02, STY-03, STY-04, STY-05; RESEARCH.md token mapping table.

## Migration-Specific Visual Contracts

Phase 6 removes `src/style.css`. The following elements have intricate styling that must be replicated exactly in `src/tailwind.css` before the old file is removed. These are the visual contracts that carry the highest regression risk.

### Octo animation (D-07 — highest risk)

Two keyframes must move into `tailwind.css` `@layer base`:

- `@keyframes octo-fly` — drives `[data-octo-wrap].celebrating`. Produces the 2s celebrate fly path across the screen with rotate and scale.
- `@keyframes octo-colours` — drives `[data-octo].celebrate > path:first-of-type`. Cycles fill through accent green → red → blue → purple → accent green over 4s. Uses the exact light/dark colour pairs from style.css. After migration, the start/end frame uses `var(--color-accent)`; mid-frames retain their hardcoded `light-dark()` pairs (these are intentional animation-only colours, not semantic tokens).

Reduced-motion: both animations must be suppressed by `@media (prefers-reduced-motion: reduce)`.

### Clue digit indicators (D-08 — intricate)

The `.recurring` pseudo-element (overdot above recurring decimal digits) must move to `tailwind.css` `@layer utilities`. The `.recurring::after` rule uses `position: absolute`, `inset-block-start: -0.55em`, `transform: translateX(-50%)`, `font-size: 1.4em`. This cannot be expressed as Tailwind utilities. app.ts line 116/118 renders `<span class="recurring">` — the class name stays, only the definition moves.

### Correct-guess digit boxes

`digit-correct` class will be removed. Instead, app.ts and octo.ts will add Tailwind utility classes directly to digit box elements: `bg-[rgba(46,139,87,0.12)] border-[rgba(46,139,87,0.4)] pointer-events-none`. This matches the pattern already used elsewhere in app.ts (lines 505, 526, 657).

### How-to-play modal (`[data-modal]`)

If still present in index.html, its dialog open/close styles must migrate to `tailwind.css` `@layer base` alongside the existing feedback modal styles. Pattern should match the `[data-fb-modal]` approach already in tailwind.css (opacity + scale transition).

### CSS reset

`@import "tailwindcss/preflight" layer(base)` must be added to tailwind.css before the style.css link tag is removed. This replaces the reset layer from style.css (box-sizing, margin/padding reset, font-family baseline, link reset).
