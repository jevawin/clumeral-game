export interface ActiveState {
  v: 1;                       // schema version — bump if shape changes; loadActive discards on mismatch

}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage (dlng_active) -> loadActive() | The persisted payload is fully attacker-controllable (any script or the user can edit localStorage). It crosses into app state on restore and MUST be treated as untrusted input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | Tampering | restored dlng_active payload | mitigate | loadActive validates schema version (v===1), date (===todayKey()), shape (possibles length-3 array, guesses array, activeBox number|null), and size (length cap) before returning; any failure returns null. Tests assert garbage/oversized/wrong-version/wrong-shape all yield null. |
| T-05-09 | Denial of service | malformed/oversized payload crashes loader | mitigate | JSON.parse is wrapped in try/catch returning null; an explicit max-length guard rejects oversized payloads before parse. App falls back to a fresh game rather than crashing. |
| T-05-10 | Spoofing | persisted clues forged to alter puzzle | mitigate | ActiveState deliberately carries NO clues — clues are re-fetched from the API on restore (D-06). A forged payload can only restore board UI state (eliminated digits / guesses), never the puzzle answer or clues. |
| T-05-11 | Information disclosure | stale yesterday state leaking into today | mitigate | loadActive returns null when date !== todayKey() (D-07), so a previous day's board never loads. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (RESEARCH.md Package Legitimacy Audit: zero new packages this phase). |
</threat_model>
