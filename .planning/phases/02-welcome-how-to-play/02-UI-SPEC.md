| Property | Value |
|----------|-------|
| Tool | none |

| Icon library | sprites.svg (inline SVG sprite sheet, existing) |

## Spacing Scale

Declared values (multiples of 4, Tailwind built-in scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1 / gap-1) | Icon gaps, tight inline spacing |
| sm | 8px (p-2 / gap-2) | Spacing between label and puzzle number |
| md | 16px (p-4 / gap-4) | Default vertical gap between content blocks |
| lg | 24px (p-6 / gap-6) | Gap between major sections (logo → octo, octo → subtitle, subtitle → HTP, HTP → button) |
| xl | 32px (p-8 / gap-8) | Top and bottom padding of the welcome screen container |
| 2xl | 48px (py-12) | Vertical breathing room on taller viewports (min-height guard) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Octopus element: 96px × 96px (see Typography section). Sized by SVG width/height attributes, not spacing tokens.
- Touch target for Play button: minimum 48px height (accessibility minimum, WCAG 2.5.8).
- Welcome screen container: `max-w-sm` (384px) horizontally, centred — matches existing `max-width: 30rem` layout convention from DESIGN-SYSTEM.md.

## Component Inventory

### Logo heading

- Element: `<h1>`
- Classes: `text-3xl font-bold text-text tracking-tight`
- Content: "Clumeral"
- No logo asset — text only.

### Octopus mascot

- Element: `<div>` containing the inline SVG copied from `index.html` lines 148–178
- SVG rendered at `width="96" height="96"` (scaled up from the existing 53×52 viewBox; keep the same `viewBox="0 0 53 52"`)
- No click interaction on the welcome screen (click-to-replay lives on the game screen only)
- `aria-hidden="true"` — decorative

### Subtitle + puzzle number

- Element: `<p>` for subtitle, `<p>` for puzzle number
- Classes: `text-base text-muted text-center`
- Subtitle content: "A daily number puzzle" (CONTEXT.md D-04)
- Puzzle number content: "Puzzle #142" format, number from `puzzleNumber()` in `src/app.ts:88` (CONTEXT.md D-05)
- Both rendered in one `<div class="text-center space-y-1">`

### How-to-play steps

- Element: `<ol>` with three `<li>` children
- Classes on `<ol>`: `text-base text-muted space-y-2 list-none`
- Classes on `<li>`: `flex gap-2` — step number in `<span class="font-semibold">` + step text in `<span>`
- No cards, no dividers, no icons, no example clue (CONTEXT.md D-07)
- Step 1: "Read the clues — each one narrows down a digit"
- Step 2: "Tap a digit box to open it, then remove digits that don't fit"
- Step 3: "When one digit remains in each box, submit your answer"
- `aria-label="How to play"` on the `<ol>`

### Play button

- Element: `<button type="button">`
- Classes: `w-full min-h-[48px] rounded-lg bg-accent text-white text-base font-semibold`
- Content: "Play" (CONTEXT.md D-12)
- On click: calls `showScreen("game")` from `src/screens.ts` (CONTEXT.md D-13)
- No loading state (puzzle pre-fetches on page load — CONTEXT.md D-13)
- `data-play-btn` attribute for DOM cache pattern

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Tap Play | `click` on `[data-play-btn]` | `showScreen("game")` — 250ms cross-fade transition |
| Page load | DOM ready | Detect `dlng_history` in localStorage; render HTP above or below Play accordingly |
| Dark mode | Inherited from Phase 1 `html.dark` class | `dark:` variants on all colour tokens apply automatically |

No hover states beyond browser defaults. No focus-visible ring beyond browser defaults (can be refined in Phase 6 polish). No animation on the welcome screen itself — transition is handled by `src/screens.ts`.

## Accessibility Contract

- Logo `<h1>` is the page's primary heading — no other `<h1>` on screen
- Octopus SVG: `aria-hidden="true"` (decorative)
- HTP `<ol>`: `aria-label="How to play"`
- Play button: visible text "Play" is sufficient — no additional `aria-label` needed
- Minimum touch target: Play button `min-h-[48px]` (WCAG 2.5.8 compliance)
- Colour contrast: `text-text` on `bg-bg` and `text-white` on `bg-accent` both pass WCAG AA at the declared hex values
